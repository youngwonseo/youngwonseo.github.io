스프링 마이크로서비스

2021-02-05 / 서영원

스프링 클라우드 컨피그

  • 마이크로서비스에서 구성 데이터를 관리
  • 애플리케이션과 구성 데이터를 완전히 분리
  • 마이크로서비스 인스턴스가 많아도 항상 동일한 구성을 유지할수 있도록 함
  • 다음과 같은 오픈 소스 프로젝트들과 통합
    • 깃
    • 콘설(Consul) : 서비스 인스턴스를 서비스에 등록하는 디스커버리 도구
    • 유레카(Eureka) : 콘설과 유사, 서비스 디스커버리를 제공

스프링 클라우드 서비스 디스커버리

  • 서비스가 배포된 물리적 위치(IP주소나 서비스 이름)를 추상화
  • 서비스 소비자는 논리적 위치를 사용해 비즈니스 로직을 콜
  • 서비스 인스턴스의 시작, 종료시점에 등록과 말소를 처리
  • 콘실, 유레카 등을 사용

스프링 클라우드/넷플릭스 히스트릭스와 리본

  • 마이크로서비스 클라이언트의 회복성 패턴을 위해 사용
  • 히스트릭스는 회로 차단기(circuit breaker), 벌크헤드(bulkhead)와 같은 서비스 클라이언트 회복성 패턴을 위한 기능 제공
  • 리본은 디스커버리 에이전트 통합 및 서비스 사용에 대한 부하 분산기능 제공

스프링 클라우드/넷플릭스 주울

  • 주울을 사용해서 라우팅 구현
  • 즉 서비스 게이트웨이를 구현할 수 있음
  • 보안, 인증, 컨텐츠 필터링, 라우팅 규칙 등을 필터로 적용가능

스프링 클라우드 스트림

  • 마이크로서비스에 경량 메시지 프로세싱을 통합 제공
  • 비동기 이벤트를 사용하는 마이크로 서비스 구축
  • RabbitMQ, Kafka와 통합기능 제공

스프링 클라우드 슬루스

  • 슬루스(sleuth)는 애플리케이션 안에서 사용되는 HTTP호출과 메시지 채널(RabbitMQ, Kafka)에 고유 추적 식별자를 통합하는 기능을 제공
  • 추적 식별자를 사용해 트렌젝셕을 추적, 로깅
  • 페이퍼트레일(Papertrail) 같은 로그 수집용 기술 도구, 집킨(Zipkin) 같은 추적 도구와 결합하여 사용
    • 페이퍼트레일: 여러 마이크로서비스의 로그를 실시간으로 질의 가능한 데이터베이스
    • 오픈집킨: 슬루스가 생성한 데이터를 사용해 단일 트랜젝션에 연관된 서비스의 호출 흐름을 시각화하는 도구

스프링 클라우드 시큐리티

  • 인증 및 인가 처리
  • 토큰 기반 사용자 검증 및 접근 권한 검증
  • JWT 지원, JWT기반 OAuth2 토큰 생성과 디지털 서명에 대한 표준 제공

프로비저닝

  • 스프링에서는 프로비저닝에 대한 기능을 제공하지는 않음
  • Travis CI, Docker등을 사용
