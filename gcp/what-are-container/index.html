컨테이너란 무엇인가?

로그인, 재고 관리, 결제, 배송등의 기능이 있는 쇼핑몰을 개발한다고 할 때 하나의 응용 프로그램을 각 모듈로 나누어 개발할 수 있습니다. 이를 실행하기 위해(또한 확장을 위해) 여러개의 소프트웨어 인스턴스가 필요한데 이때 각 인스턴스는 하드웨어(CPU, RAM, DISK, ETC) 및 소프트웨어(OS, NETWORK, ETC) 요구사항이 있습니다.

컨테이너는 이러한 요구사항(H/W, S/W, to Scale)을 코드가 아닌 외부 설정 파일(configuration files)을 통해 선언하여 환경(개발/운영)을 쉽게 관리할 수 있는 이점을 줍니다.

왜 가상화된 운영체제를 통해 독립된 컨테이너를 실행하는게 중요한지 설명합니다. 개발, staging, 제품등 다른 환경에서의 continuous delivery를 위한 준비작업을 합니다.

Introduction to Containers and Docker Part1 : Containers

기존방식

베어 메탈 서버같은 하드웨어를 설치하고 OS 또는 커널, 종속되는 소프트웨어 및 환경설정 후 응용프로그램을 설치 Bare Metal Server :

하이퍼 바이저

VMWare가 개발한것으로 하드웨어를 커널과 분리(하드웨어와 응용 분리가 목적)하는 계층, 이것은 추상화 레벨1에 해당(Virtual Machine) 그러나 하나의 VM에 동일한 어플리케이션 또는 여러버전의 어플리케이션을 설치 할 수 없는 문제가 존재 이것을 여러개의 VM설치하여 해결, 그러나 이것은 비용이 많이들고 비효율적

이것을 해결한것이 다음 그림 하드웨어와 하이퍼 바이저를 분리, 컨테이너를 사용하여 응용 및 종속성을 분리 portable, 효율적, Container 내부는 커널이나 하드웨어와 종속성이 없음
