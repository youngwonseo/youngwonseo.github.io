자바스크립트 Promise

05 Apr 2018    ·   

Promise

1. Promise의 탄생

비동기 대한 디자인 패턴 Promise!!

  • 자바스크립트는 단일쓰레드 기반에서 동작한다.
    • 실제 모든 작업이 단일큐에 들어간다.
    • 파일에 대한 I/O가 필요한 경우 I/O가 실행되는 동안 모든 작업이 멈춘다!
      • 이것을 blocking 된다라고 표현한다.
    • 하지만 우리는 특정 I/O가 발행되는 시점에서도 사용자 레벨의 코드가 멈추기를 원하지 않는다.
  • 그래서 비동기 작업이 필요한 시점에는 콜백(이벤트 등)을 이용해 왔다(아래는 몇 가지 예시이다).
    • 비동기 통신이 필요한경우 통신완료 시 실행될 콜백함수를 작성하고 ajax를 호출한다.
      • 통신이 완료되면 정의해놓은 콜백함수가 실행된다.
    • addEventListener등을 통해 특정 이벤트에 콜백함수를 정의해 놓으면 해당 이벤트가 발생할 때 콜백함수가 실행된다.
      • DOM Load등
  • 이러한 사용에는 몇 가지 문제점이 있다.
    • 비동기가 많아질수록 이해하기 힘든 코드가 작성된다.
    • 콜백에 콜백에 콜백이 추가되는 코드를 작성하기도 한다.
      • 콜백이 완료된 후 또 다른 콜백이 필요한 경우
      • 이를 콜백지옥이라고 부른다(http://callbackhell.com/).
    • 에러를 감지하지 못한다.
      • 비동기 처리(I/O)중에 발생한 에러(reject, timeout 등)는 api에서 어느정도 처리하지만 콜백함수안에서 에러가 발생한 경우는 문제가 있다.
  • 그렇다! 이러한 비동기에 대한 문제점을 해결하기 위한 것이 Promise이다!
    • 통일된 비동기 처리방식
    • 이해하기 쉬운 비동기 루틴
    • 비동기의 성공/실패를 감지
    • 기존에는 여러 라이브러리를 통해 Promise를 구현했지만 지금은 브라우에서 제공!
      • 지원안하는 경우 polyfill 사용

2. Promise 용어

  • Promise에 대한 기본용어는 다음과 같다. 매우 간단하다!
    • 처리됨(fulfilled) - 작업성공
    • 거부됨(rejected) - 작업실패
    • 보류됨(pending) - 처리, 거부 둘다 안됨
    • 해결됨(settled) - 처리 또는 거부됨

3. Promise의 기본적인 사용법

  • 그럼 프로미스의 사용법을 살펴보자. 간단한 예제를 살펴보며 위의 용어들이 어떻게 사용되는지 파악하자. 
    //프로미스 객체 생성
var promise = new Promise(function(resolve, reject) {
  // do a thing, possibly async, then…
  // async!!
      
  if (/* everything turned out fine */) {
    resolve("Stuff worked!");
  } else {
    reject(Error("It broke"));
  }
});

//프로미스 사용
promise.then(function(result) {
  console.log(result); // "Stuff worked!"
}, function(err) {
  console.log(err); // Error: "It broke" 
});
  • 먼저 Promise객체를 생성하는데 이때 생성자를 매개변수로 정의한다.
    • 생성자는 resolve와 reject를 받고 작업성공시 reslove, 실패시 reject를 호출한다
      • 작업성공시 resolve가 아닌 단순값 반환도 가능하다.
    • 이때 반환할 값을 매개변수로 넘겨준다.
    • 중요한 점은 Promise객체안의 동작이 비동기 적으로 이루어진다.
  • 생성된 객체를 실행(호출)하기 위해 객체의 .then을 호출한다.
    • 성공시와 실패(에러발생)시의 콜백함수를 정의한다.
      • 성공시 호출되는 함수 - onFulfilled, 실패시 호출되는 함수 - onRejected 라 한다.
      • onRejected는 옵션입니다(즉 생략가능). 실패를 처리하는 또 다른 방법(좀 더 보편적인방법- .catch)은 다음과 같다. 
        promise.then(function(result) {
  console.log(result); // "Stuff worked!"
}).catch(function(err) {
  console.log(err); // Error: "It broke" 
});
    • 객체 생성시점에서 정의한 resolve와 reject의 반환이 성공 또는 실패함수의 호출을 결정한다.
      • resolve -> onFulfilled
      • reject -> onRejected
    • then과 catch는 chaning이 가능하다.
      • 이것은 비동기 작업단위(모듈)들의 처리순서를 보장받을 수 있다는 것이다.
  • 위의 Promise형식은 무조건적이 아닌 아주 간단한 예제일뿐이다.
    • 실제 구현시에는 아주 다양한 형태로 구현된다.
  • 그러면 여러가지 예제를 통해 사용법을 익혀보자.

4. Promise API

  • Promise.resolve, Promise.reject
    • Promise.resolve - new Promise를 단축표기한며 then을 실행(콜백호출)한다.
    • Promise.reject - new Promise를 단축표기한며 catch을 실행(콜백호출)한다. ```javascript new Promise(function(resolve){ resolve(1234) }).then(function(value){ console.log(value) })

    Promise.resolve(1234).then(function(value){ console.log(value) }) ```

    • 이전에 말한것처럼 단순값(1234)를 반환한다.
      • promise객체 초기화나 테스트코드를 작성될 때 활용할 수 있다.
    • thenable객체를 promise로 반환할 수 있다.
      • 즉 then을 가지는 유사 promise객체를 promise로 처리할 수 있다.
      • 이는 thenable을 지원하는 여러 third-party library의 비동기처리를 보다 쉽고 명확한 코드로 작성할 수 있게 만든다!
  • Promise.prototype.then, Promise.prototype.catch
    • then, catch는 Promise객체로 부터 사용할 수 있다.
    • 비동기 처리의 실행 순서를 보장받을 수 있다.
    • 메서드 체인을 가능하게 하고 예외처리를 원하는 시점에 콜백시킬 수 있다. 
      promise
    .then(taskA)
	.then(taskB)
	.catch(errorA)
	.then(taskC)
	.catch(errorB)
	.then(taskD)
    • 위의 코드를 해석하며 콜백시점 및 예외처리에 대한 보장을 살펴보자.
      • 실행시점이 taskA -> taskB-> taskC- > taskD로 보장받는다.
      • taskA와 taskB에서 예외발생시 errorA, taskC에서 예외발생시 errorB에서 처리된다.
      • taskA또는 taskB에서 예외가 발생하여 errorA가 호출되도 다시 taskC는 호출된다.
      • 즉 비동기에 대한 순서를 정의하고 예외처리 시점을 명시적으로 알 수 있다.

  • Promise.all, Promise.race
    • Promise.all - 여러개의 Promise가 모두 fulfilled될 경우 then을 호출
    • Promise.race - 여러개의 Promise중 하나만 fulfilled되면 then 호출
    • 이해하기 딱히 여려울 것이 없다! 
      var promise1 = new Promise(function(resolve, reject) {
    ...
})

var promise2 = new Promise(function(resolve, reject) {
    ...
})

//All
Promise.all([promise1,promise2]).then(function(){
    ..
}).catch(function(){
    ..
});

//Race
Promise.rece([promise1,promise2]).then(function(){
    ..
}).catch(function(){
    ..
});

5. Promise 특징

항상 비동기실행된다.

  • 동기와 비동기에 대한 선택이 없다.
  • 이로인해 동기와 비동기가 혼재했을 때 발생하는 문제를 예방한다.

한번만 실행된다.

  • addEventListener등을 통해 등록된 콜백은 해당 이벤트가 발생할때마다 실행된다.
  • Promise는 오직 한번만 실행될 뿐이다.
