이미지 로딩 속도 최적화

이미지 로딩 최적화를 위한 방법들로 다음과 같은것들이 존재한다. 이미지 로딩 최적화의 경우 개부분 자바스크립트를 사용하지 않고 구현가능하다.

  • CSS 스프라이트 : 이미지 파일을 하나로 합쳐 다운로드하여 불필요한 request를 줄이는 방법 예) [앞으로],[뒤로] 등의 버튼에 대한 이미지를 하나로 합쳐 하나의 이미지에 대해 background-position 으로 출력할 위치를 정하면 이미지에 대한 request가 한번만 발생하게 만드는것

  • 이미지 파일의 메타 정보 제거 :

  • display:none 이미지 처리 : 자바스크립트를 이용한 방법으로 display:none으로 처리될 이미지들은 처음부터 다운로드하지 않고 display가 none이 아닌 항목(inline, block 등)으로 변경될 때 동적으로 이미지 태그를 추가하는 방법

    function show_image(){
    var image = document.createElement('img');
    image.src = './image.png';
    document.getElementById('#parent').appendChild(image);
}

다이내믹 로딩을 활용한 최적화

먼저 다이내믹 로딩에 대해 용어정리를 하자.

  • 다이내믹 로딩 : 지연로딩과 온디맨드 로딩을 포함해 동적인 로딩을 총칭하는 용어
  • 지연로딩 : 웹 페이지 구성요소를 다운로드하는 시점을 의도적으로 늦추는 방법을 가르키는 용어
  • 온디맨드 로딩 : 사용자가 원하는 시점에 웹 페이지 구성 요소를 다운로드하는 방법을 가리키는 용어

지연로딩

지연로딩은 렌더링 속도 향상에 중점을 둔 성능 최적화 방법으로, 웹 페이지 구성 요소들의 다운로드 시점을 조절하여 렌더링 속도를 높이는 방법이다. 다음은 대표적인 두 가지의 웹 로딩(구성요소 다운로드)에 대한 순서이다.

  • HTML > 스타일시트 > 자바스크립트 > 이미지
  • HTML > 스타일시트 > 이미지 > 자바스크립트

위 두 유형중 어느것이 빠를까? 아래항목 더 빠르다. 이유는 위의 경우 자바스크립트를 파싱하고 실행하는 동안에는 블로킹(자바스크립트에 대한 독립적이고 안정적인 실행을 보장하기위해)이 발생해 다른작업과 병행하지 않으므로 이미지 다운로드가 지연되기 때문이다.

그렇다면 아래의 유형을 구현 방법은 어떻게 될까? 방법은 간단하다! 그것은 스크립트에 대한 코드를 body의 마지막에 두는것이다.

<html>
    <body>
    	...
		<script src="..."></script>
		<script src="..."></script>
    </body>
</html>

온디멘드 로딩

온디멘드는 사용자의 요구사항이 발생하면 필요한 요소들을 다운로는 하는 최적화 방식이다. 요소들에는 이미지나 텍스트가 될 수도 있지만 여기서는 자바스크립트 소스코드 자체를 예로 든다. 즉 자바스크립트를 동적으로 로딩하는 방식으로 다음과 같은 방법들이 존재한다.

  • XHR 기술들을 활용한 자바스크립트 동적 다운로드
  • Script DOM Injection 방식을 활용한 자바스크립트 동적 다운로드
  • IFrame을 활용한 자바스크립트 동적 다운로드
  • document.write() 메서드를 활용한 자바스크립트 동적 다운로드

이 네 가지 방법은 브라우저의 자바스크립트 블로킹을 최소화하면서 자바스크립트를 다운로드 하는 방식으로 script태그를 활용해 정적으로 추가하는 방식이아닌 비동기식, 즉 동적으로 다운로드하는 것이 핵심원리이다. 즉 사용하지 않는 기능의 스크립트들을 미리 다운로드 하는것이 아닌 필요할 경우 다운로드하고 사용하는 것이다. 그럼 여기서 하나씩 살펴보자. 여기서는 핵심적인 부분과 이에 해당하는 코드위주로 설명하니 자세한 설명은 책을 참조하길 바란다.
