스프링 클라우드 컨피그

2021-02-06 / 서영원

기존에 존재했던 저수준의 프로퍼티 파일(YAML, JSON, XML등)은 수백 개의 마이크로서비스와 수많은 마이크로서비스 인스턴스가 실행되는 클라우드 환경에서는 적합하지 않음

  • 깃
  • 로컬 파일 시스템
  • 하시코프 볼트

  • JDBC 데이터베이스

  • 배포되는 코드와 설정정보들을 완전히 분리
  • 서버 및 애플리케이션을 빌드하고 배포하는 과정에서 변경되지 않는 이미지를 빌드
  • 서비스가 시작할때 마이크로서비스가 읽어 올 수 있는 중앙 저장소를 이용해 애플리케이션 구성 정보를 주입

애플리케이션 구성관리 4가지 원칙

  • 분리 : 실제 물리적인 서비스의 배포와 구성 정보를 완전히 분리, 즉 애플리케이션에 구성정보가 포함되어있으면 안됨
  • 추상화 : 구성데이터를 파일이나 JDBC등을 사용한 직접적인 접근보다 RESTful로 Json과 같은 형태로 조회되도록 해야함
  • 중앙 집중화 : 구성데이터를 가능한 소수의 저장소에 집중화함
  • 견고성 : 구성데이터를 서비스와 완전히 분리하고 중앙 집중화하므로 어떤 솔루션을 사용하더라도 고가용성과 다중성을 구현할 수 있어야함

구성 관리 시스템 선택하상

  • Etcd
  • Eureka
  • Consul
  • Zookeeper
  • Spring Cloud Config Server
