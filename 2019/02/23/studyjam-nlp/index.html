구글 NLP 스터디잼 - API소개

23 Feb 2019    ·   

이번에 구글에서 진행하는 스터디잼 진행에 대한 정리글 입니다. 다음과 같은 4가지 과정을 quicklab을 통해 수행하였습니다.

  • Google Cloud Speech API: Qwik Start
  • Cloud Natural Language API: Qwik Start
  • Speech to Text Transcription with the Cloud Speech API
  • Entity and Sentiment Analysis with the Natural Language API

이번 스터디잼을 이수한 결과 사실상 대부분이 API호출이었습니다. CURL을 통하여 json으로 구성된 config파일을 사용하여 NLP API를 호출하고 호출결과를 json으로 응답받는 것입니다.

Google Cloud Speech API: Quick Start

Google Cloud Speech는 Speech-to-Text를 제공하는 API입니다. 즉 오디오에 포함된 언어를 recognition하여 텍스트로 변환하는 기능을 제공합니다. API를 사용하기 위해 GCP 콘솔에서 API키를 생성합니다.

다음 키를 API_KEY라는 환경변수로 할당합니다. 이것은 리눅스를 사용할 경우 이고 API를 호출할때 키를 직접 입력해도 상관없습니다.

export API_KEY=<YOUR_API_KEY>

다음은 요청한 config입니다.

request.json

{
	"config": { 
		"encoding":"FLAC", 
		"sample_rate": 16000, 
		"language_code": "en-US" 
	},
	"audio": {
		"uri":"gs://cloud-samples-tests/speech/brooklyn.flac" 
	} 
}

각 항목에 대한 설명입니다.

  • encoding : speech에 대한 파일 인코딩
  • sample_rate : 오디오 데이터의 Hertz

위 두 항목은 필수 파라미터입니다. 그 외 다른 옵션에 대해서는 문서 를 확인해 주세요. audio의 uri에 text로 변환할려고하는 음성파일이 존재합니다. 음성파일은 여기서 들을 수 있습니다.

다음은 실제 API호출입니다. curl 명령어를 통해 api서버로 요청을 보내고 json포맷으로 응답을 받습니다. ${API_KEY}는 위에서 설정한 환경변수입니다.

curl -s -X POST -H “Content-Type: application/json” –data-binary @request.json \ “https://speech.googleapis.com/v1beta1/speech:syncrecognize?key=${API_KEY}”

응답결과는 다음과 같습니다.

{  
	"results":  [  
		{  
			"alternatives":  [  
				{  
					"transcript":  "how old is the Brooklyn Bridge",  
					"confidence":  0.98267895  
				}  
			]  
		}  
	]  
}

transcript가 변환된 텍스트이고 confidence가 정확도입니다. 위에서 curl을 통해 api를 요청할 때 syncrecognize라고 명시된 부분이 있었습니다. speech api는 동기와 비동기 모두를 지원합니다. 즉 하나의 파일로 구성된 파일이 아니라 스트리밍을 통한 오디오가 생성되는 동안에도 text로 변환하는 기능을 제공합니다. 스터디잼에서는 이 부분은 진행되지 않았지만 문서를 참조하면 async하게 사용하는 방법을 학습할 수 있습니다.

Cloud Natural Language API: Quick Start

두 번째는 자연어처리 API입니다. 사람의 말에 대해 semantic한 부분을 recognition해주는 API입니다. 챗봇 등의 어플리케이션을 구현할때 사용하면 유용할 것 입니다. 한 가지 명심해야 할 부분은 gcloud 콘솔에서 실습이 진행됩니다.

먼저 구글 클라우드 프로젝트 키를 환경변수에 할당합니다.

export GOOGLE_CLOUD_PROJECT=$(gcloud config get-value core/project)

다음 NL API의 계성 생성

gcloud iam service-accounts create my-natlang-sa \ –display-name “my natural language service account”

다음 위에서 생성된 계정을 통해 credentials을 생성합니다. 해당 credentials은 key.json파일로 생성됩니다.

gcloud iam service-accounts keys create ~/key.json \ –iam-account my-natlang-sa@${GOOGLE_CLOUD_PROJECT}.iam.gserviceaccount.com

해당 key파일의 위치를 환경변수에 할당합니다.

export GOOGLE_APPLICATION_CREDENTIALS=”/home/USER/key.json”

마지막으로 실제 api를 호출합니다.

gcloud ml language analyze-entities –content=”Michelangelo Caravaggio, Italian painter, is known for ‘The Calling of Saint Matthew’.”

다음은 응답으로 출력되는 분석결과입니다.

		"type": "PERSON",

			"wikipedia_url": "http://en.wikipedia.org/wiki/Italy"
		},
		"salience": 0.13870546,
		"mentions": [{
			"text": {
				"content": "Italian",

  • name : entity 이름
  • type : entity 타입, person ? location ? event ? etc..
  • metadata : 관련 위키피디아 URL
  • salience : 해당 entity가 전체 텍스트에서 나타내는 중요도를 [0,1]로 표현
  • mentions : 입력된 문장에서 동일한 의미를 가지는 키워드, (예, "나는 사람이다" 에서 “나”는 “사람”과 같은 의미)

Speech to Text Transcription with the Cloud Speech API

첫 번째 실습과 거의 유사한 API입니다. speech를 text로 변경하는데 옵션이나 사용방법이 거의 유사합니다.

첫 번째의 실습과 같이 API KEY를 먼저 발급받습니다. 그 후 환경변수에 할당합니다.

export API_KEY=<YOUR_API_KEY>

오디오파일을 text로 변경합니다.

request.json

{
	"config": {
		"encoding": "FLAC",
		"languageCode": "en-US"
	},
	"audio": {
		"uri": "gs://cloud-samples-tests/speech/brooklyn.flac"
	}
}
  • encoding : 파일 인코딩
  • languageCode : 언어코드

encoding은 필수 파라미터 입니다. 더 많은 정보는 documentation를 확인해주세요

다음은 실제 API호출입니다.

curl -s -X POST -H “Content-Type: application/json” –data-binary @request.json \ “https://speech.googleapis.com/v1/speech:recognize?key=${API_KEY}”

{
	"results": [{
		"alternatives": [{
			"transcript": "how old is the Brooklyn Bridge",
			"confidence": 0.98267895
		}]
	}]
}

다음은 불어에 대한 요청과 응답입니다. 오디오파일은 여기서 들을 수 있습니다.

{
	"config": {
		"encoding": "FLAC",
		"languageCode": "fr"
	},
	"audio": {
		"uri": "gs://speech-language-samples/fr-sample.flac"
	}
}

{
	"results": [{
		"alternatives": [{
			"transcript": "maître corbeau sur un arbre perché tenait en son bec un fromage",
			"confidence": 0.9710122
		}]
	}]
}

Entity and Sentiment Analysis with the Natural Language API

이번 실습에서는 주어진 문장으로 부터 entity를 추출하고 sentiment와 syntactic을 분석 그리고 category별로 분류를 수행합니다(두 번째 실습과 유사). 즉 Natual Language API를 사용하여 linguistic analysis를 수행한다고 할 수 있습니다.

이전 실습과 유사하게 API키를 생성하고 환경변수로 설정합니다.

export API_KEY=<YOUR_API_KEY>

먼저 NL API를 사용해서 분석할 파일을 생성합니다. json으로 구성되어 있으며 분석할 text가 포함되어있습니다.

request.json

{
	"document": {
		"type": "PLAIN_TEXT",
		"content": "Joanne Rowling, who writes under the pen names J. K. Rowling and Robert Galbraith, is a British novelist and screenwriter who wrote the Harry Potter fantasy series."
	},
	"encodingType": "UTF8"
}
  • type : 분석할 텍스트 타입(PLAIN_TEXT or HTML)
  • content : 분석할 텍스트, 필요에 따라 Cloud Storage의 파일을 바로 분석할 수 있습니다(gcsContentUri)
  • encodingType : 텍스트의 인코딩 타입

curl을 이용하여 api를 호출합니다.

curl “https://language.googleapis.com/v1/documents:analyzeEntities?key=${API_KEY}” \ -s -X POST -H “Content-Type: application/json” –data-binary @request.json

위 request.json에 대한 분석 결과는 다음과 같습니다.

		},
		"salience": 0.7980405,
		"mentions": [{
			"text": {
				"content": "Joanne Rowling",
				"beginOffset": 0
			},
			"type": "PROPER"
		}, {
			"text": {
				"content": "Rowling",
				"beginOffset": 53
			},
			"type": "PROPER"
		}, {
			"text": {
				"content": "novelist",
				"beginOffset": 96
			},
			"type": "COMMON"
		}, {
			"text": {
				"content": "Robert Galbraith",
				"beginOffset": 65
			},
			"type": "PROPER"
		}]
	}, 
	...
  ]
}

이번 실습은 두 번째 실습에 대한 설명과 비슷합니다.

  • type : entity의 type
  • salience : 해당 entity가 전체텍스트에서의 중요도를 [0,1]로 표현
  • mentions : 주어진 텍스트에서 해당 entity와 동일한 의미를 나타내는 텍스트들
{
	"document": {

Sentiment analysis with the Natural Language API

이번에는 sentiment analysis에 대한 실습을 수행해 봅니다. 먼저 다음과 같이 요청파일을 생성합니다.

request.json

 {
 	"document": {
 		"type": "PLAIN_TEXT",
 		"content": "Harry Potter is the best book. I think everyone should read it."
 	},
 	"encodingType": "UTF8"
 }

curl을 사용하여 api를 호출합니다. 앤드포인트가 analyzeSentiment입니다.

curl “https://language.googleapis.com/v1/documents:analyzeSentiment?key=${API_KEY}” \ -s -X POST -H “Content-Type: application/json” –data-binary @request.json

응답 json입니다.

{
	"documentSentiment": {
		"magnitude": 0.8,
		"score": 0.4
	},
	"language": "en",
	"sentences": [{
		"text": {
			"content": "Harry Potter is the best book.",
			"beginOffset": 0
		},
		"sentiment": {
			"magnitude": 0.7,
			"score": 0.7
		}
	}, {
		"text": {
			"content": "I think everyone should read it.",
			"beginOffset": 31
		},
		"sentiment": {
			"magnitude": 0.1,
			"score": 0.1
		}
	}]
}
  • score : 부정(-1.0)과 긍정(1.0)의 지수, 0은 중립
  • magnitute [0, infinity] - positive, negative와 관계없이 문상에서의 sentiment의 weight, 즉 감정의 크기를 0에서 무한대로 표현, 0은 중립(보통의 상태)

또 다른 entity sentiment analysis에 대한 실습니다. 요청 파일입니다.

이번에는 좀더 디테일한 linguistic analysis api입니다. 상세한 semantic and syntactic elements of the text 결과를 제공합니다.
요청파일입니다.

reqeuest.json

{
	"document": {
		"type": "PLAIN_TEXT",
		"content": "Joanne Rowling is a British novelist, screenwriter and film producer."
	},
	"encodingType": "UTF8"
}

api호출입니다.

curl “https://language.googleapis.com/v1/documents:analyzeSyntax?key=${API_KEY}” \ -s -X POST -H “Content-Type: application/json” –data-binary @request.json

결과입니다.

{
	"text": {
		"content": "is",
		"beginOffset": 15
	},
	"partOfSpeech": {
		"tag": "VERB",
		"aspect": "ASPECT_UNKNOWN",
		"case": "CASE_UNKNOWN",
		"form": "FORM_UNKNOWN",
		"gender": "GENDER_UNKNOWN",
		"mood": "INDICATIVE",
		"number": "SINGULAR",
		"person": "THIRD",
		"proper": "PROPER_UNKNOWN",
		"reciprocity": "RECIPROCITY_UNKNOWN",
		"tense": "PRESENT",
		"voice": "VOICE_UNKNOWN"
	},
	"dependencyEdge": {
		"headTokenIndex": 2,
		"label": "ROOT"
	},
	"lemma": "be"
}
  • partOfSpeech : 해당 text가 명사인가, 동사인가를 판단

  • dependencyEdge : dependency parse tree의 생성 결과, sentence에 포함된 각 word들이 어떤 relation을 갖고 있는지 표현

  • headTokenIndex : 해당 text의 인덱스(sentense도 배열이므로 0부터 시작)
  • lemma : 입력된 동사에 대한 일반동사(the words run, runs, ran, and running all have a lemma of run)

Overview
