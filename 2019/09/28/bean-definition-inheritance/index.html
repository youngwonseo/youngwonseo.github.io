Bean Definiton Inheritance

1.7. Bean Definition Inheritance

  • Bean 정의시 많은 설정정보가 포함될 수 있음
    • constructor arguments
    • property values
    • container specific information
      • initialization method
      • static factory method name
  • 하위Bean은 상위Bean을 상속
  • 하위Bean은 필요에 따라 상위속성에 대한 override 가능
  • 상위Bean과 하위Bean을 통해 많인 typing으로 정의가능

  • 이것은 효과적으로 템플릿한 것

  • AplicationContext 인터페이스를 programmatically하게 사용할 경우, 하위 Bean은 ChildBeanDefinition으로 표현됨
  • 대부분의 사용자는 해당수준에서 사용하지 않음
  • 대신 ClassPathXmlApplicationContext와 같은 선언적 정의 사용
  • XML기반의 설정을 사용할때, 상위Bean 속성을 사용하여 하위Bean을 정의가능
    • 상위Bean의 속성을 을 명시
  • 다음은 거기에 대한 예제
<bean id="inheritedTestBean" abstract="true"
        class="org.springframework.beans.TestBean">
    <property name="name" value="parent"/>
    <property name="age" value="1"/>
</bean>

<bean id="inheritsWithDifferentClass"
        class="org.springframework.beans.DerivedTestBean"
        parent="inheritedTestBean" init-method="initialize">  
    <property name="name" value="override"/>
    <!-- the age property value of 1 will be inherited from parent -->
</bean>

https://docs.spring.io/spring/docs/current/spring-framework-reference/core.html#beans-child-bean-definitions

  • 하위Bean의 정의시 구체적인 정의가 존재하지 않고 ovveride도 하지 않으면 상위Bean정의를 사용
    • 이경우 하위Bean은 상위Bean과 호완가능해야함(property value)
  • 하위Bean정의는 새로 추가되는 값과함께 다음과 같은 항목을 상위Bean에서 상속
    • scope
    • constructor argument values
    • property values,
    • override된 method
  • 다음과 같은 setting또한 override된다
    • any scope
    • initialization method
    • destory method
    • stataic factory method
  • 다음과 같은 나머지 설정은 하위에서 정의
    • depends on
    • autowired mode
    • dependency check
    • singleton
    • lazy init
  • 이전의 예제는 abstract속성의 abstract를 사용하여 상위Bean정의를 명시적으로 표시
  • 상위Bean 정의에서 클래스가 기술되지 않으면 명시적으로 상위빈을 정의를 명시해야함
  • 다음은 거기에 대한 예제
<bean id="inheritedTestBeanWithoutClass" abstract="true">
    <property name="name" value="parent"/>
    <property name="age" value="1"/>
</bean>

<bean id="inheritsWithClass" class="org.springframework.beans.DerivedTestBean"
        parent="inheritedTestBeanWithoutClass" init-method="initialize">
    <property name="name" value="override"/>
    <!-- age will inherit the value of 1 from the parent bean definition-->
</bean>
  • 상위Bean은 불완전하기 때문에 인스턴스화 할 수 없음
  • 또한 명시적으로 abstract라는것을 표기해야함
  • 정의가 abstract일 경우, 하위Bean을 위한 pure template bean 정의로 사용될 수 있음
  • abstract 상위 Bean을 다음과 같이 사용할려고 할때 오류가발생
    • 다른 Bean의 ref property로 사용
    • 명싲거으로 getBean()호출
  • 유사하게 컨테이너 내부의 preInstantiateSingletons() 메서드는 abstract로 정의된 Bean정의를 무시함
