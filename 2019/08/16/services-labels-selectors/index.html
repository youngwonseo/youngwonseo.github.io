Services, Labels, and Selectors

  • service는 POD에 고정 IP를 할당, POD간 통신 가능
  • cluster에는 외부 IP와 내부 IP가 존재하며, 포트 노드는 IP에 대한 접근을 인식
  • IP 접근시 로드 벨런스 사용
  • 다른 설정의 여러개의 서비스운영
  • 위 그림에서 프론트 및 백엔드 존재
  • 구글 클라우드 로드벨런서 작동

  • Service를 정의하는 yaml
  • type이 LoadBalancer

  • Labels은 API object에 할당하여 identity를 표시하는 메타데이터
  • pod를 위한 그룹화
  • 검색 및 필터링
  • 논리적으로 labels을 사용하면 pod의 작동방식별로 빠르게 식별할 수 있음

  • 위 그림에서 4개의 pod, 3개의 label이 존재
  • 만약 9000개의 컨테이너에서 특정 요소를 찾을때 label을 이용하면 쉽게 찾음
  • 응용을 labels에 매핑할 수 있음
  • Tag, lable 사용

  • kubernetes는 모든것을 체크
  • response가 없어나 부정적이면 pod들 다시 실행
