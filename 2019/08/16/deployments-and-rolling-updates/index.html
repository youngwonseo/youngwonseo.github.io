Deployments and Rolling Updates

16 Aug 2019    ·   
  • 컨테이너 기반의 장점은 응용을 분리해서 서비스로 연결하는 것
  • 또 다른 장점은 개발, 스테이징, 제품환경의 각 환경들을 비슷하게 유지하는것, 그래서 마이그레이션 쉬움
  • 이를 위해 kubernetes는 deployments, replica sets이라는 도구를 제공해서 빌드 및 운영환경을 나누고 스케일하는것을 관리하게 함
  • 이러한 툴을 사용하면 rolling updates, canaries, blue greens 업데이트를 쉽게할수 있음

  • deployment는 POD를 관리하고 실행하기 위해 ReplicaSets에 의존적
  • yaml파일에 작성한 동작 방식, 원하는 개수 등으로 POD의 수와 상태가 유지

  • ReplicaSet을 4로 지정

  • replicas가 3일때 1개가 실패(장애발생)하면 하나를 생성

  • 3개의 POD 또는 3개의 replica를 실행하는 것이 좋음

  • Rolling updates를 사용하면 중단없이 이미지버전을 다른 버전으로 점진적으로 업데이트 가능

  • roll out없이 roll over되며 업데이트 가능
  • hello1 POD를 하나씩 종료하고 새로운 ReplicaSet에 hello2를 생성
  • 프로덕션중에 수행
