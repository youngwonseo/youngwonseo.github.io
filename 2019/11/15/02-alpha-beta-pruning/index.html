Alpha-beta Pruning

15 Nov 2019    ·   

*Minmax Search를 먼저 이해하여야 한다.

Alpha Beta Pruning 이란 Minmax Search에서 발생하는 연산을 최소한으로 감소시킬수 있는 알고리즘이다. 기존의 MinmaxSearch는 트리를 생성한 후 상태를 평가한다. 즉 생성과 판단이 분리되어 있고 각각 이루어 져야하지만 Alpha Beta Pruning은 이러한 Minmax의 문제점을 보완하고자 생성과 판단을 동시에 진행한다. 이는 트리를 깊이 생성할수록 생성되기전 Pruning으로 인하여 연산시간이 훨씬 감소한다.

그림1 은 Minmax Search에서 보인 그림이다. 틱택톡 게임의 시작상태로써 시작노드로 부터 한수 앞을 내다보도록 계산하기 위해 트리의 깊이 2까지의 모든 경우의 수를 계산한다. 그림 2는 Aplha Beta의 예로서 시작 노드가 Max노드이고 A,B는 MIN노드이다(그림1의 일부분이다).

A노드를 먼저 살펴보자. A의 자식의 결과 A가 가지는 값은 -1로 지정되었다. 이는 MIN노드가 자식값 중 최소값을 가질 수 있으므로 당연한 결과이다. 이제 B노드를 생성하기전 시작노드를 살펴보자. 시작노드는 자식의 값중 최대값을 가져야 하는데 첫번째 자식의 결과로서 -1이라는 값을 반환 받았고 이는 시작노드가 가질수 있는 최소의 값이다(자식의 값중 최대값을 가질수 있으므로 -1 보다 작을 경우는 버리기 때문에). 여기서 시작노드의 최소값으로 -1, 즉 알파값는 해당 노드의 하한값을 나타내고 시작노드의 알파값이 -1이 되었다.

이제 B노드의 값을 계산해보자. B의 첫번째 자식노드인 C의 값이 -1이 되었다. B는 노드는 자식 중 최소값을 가져야 하므로 C값 -1이 계산된 시점에 B의 값은 -1보다 같거나 작은 값을 가질수 있다는 것을 알수있다. 이때 B의 상한값이 -1이 되는데 이것이 배타값을 나타낸다.

이 시점에서 정리해보면 B의 값은 -1보다 큰 값을 가질 수 없다. 하지만 시작노드는 -1보다 큰 값을 가질수 있다. 즉 B의 다른 자식노드를 계산해봤자 B는 A보다 선호될 수는 없다(정확하게 말하면 값이 같을 수는 있어서 같은 선호도를 가질 수는 있지만 더 좋은 선호도를 가질수는 없다). 이로써 B의 계산이 끝나게되는데 이는 기존의 Minmax Search에서 모든 경우의 수를 계산하는 것에 비해 연산수가 확실히 줄어든다(특히 더 앞의 수를 바라볼수록 계산량이 훨씬 줄어든다).

function alphabeta(node, depth, α, β, maximizingPlayer)
      if depth = 0 or node is a terminal node
          return the heuristic value of node
      if maximizingPlayer
          v := -∞
          for each child of node
              v := max(v, alphabeta(child, depth - 1, α, β, FALSE))
              α := max(α, v)
              if β ≤ α
                  break (* β cut-off *)
          return v
      else
           v := ∞
           for each child of node
              v := min(v, alphabeta(child, depth - 1, α, β, TRUE))
              β := min(β, v)              
              if β ≤ α
                  break (* α cut-off *)
           return v
 
(* Initial call *)
alphabeta(origin, depth, -∞, +∞, TRUE)
